What is Placard?

An HTML + CSS renderer that outputs a plain image, not a web page.

HTML & CSS, for real

A hand-written tokenizer, DOM, cascade, and box-model layout engine — margin, border, padding, content, laid out exactly like this.

Live data, static pixels

Add a data-preset or data-connector attribute and the value gets fetched and baked into the image at render time.

Real CSS color syntax

Hex, rgb(), rgba(), and named colors all parse to the same internal color type.

PNG & WebP output

Every render encodes straight to a raster image — pick the format with the extension on the URL, or the flag on the CLI.

CLI & HTTP API

Render from the command line, or hit /r/<payload> from any app, README, or <img> tag.

Connectors: live data, static pixels

This is what makes Placard more than a renderer: an element's text can be resolved from a live network call at render time, then baked permanently into the image that comes back. The HTML describes where the value comes from; the PNG that comes out is a plain static image with that value already drawn in — nothing about it depends on JavaScript running wherever it's displayed.

Two ways to pull in data

Attribute Behavior
data-connector="<url>" Fetches the URL, decodes the body as UTF-8, trims it, and uses it verbatim as the element's text — for any endpoint that already returns exactly the text to display.
data-preset="<name>" Selects one of Placard's built-in resolvers, which knows the shape of a specific upstream API and how to turn its JSON response into a value. Other data-* attributes on the same element become that resolver's parameters.

Placard bundles well over a hundred presets across 140+ upstream services. Full https://github.com/badges/shields API coverage too! 

<span
    data-preset="github-stars"
    data-owner="rust-lang"
    data-repo="rust"
>0</span>

The number inside the tag is a fallback: if the fetch fails for any reason — bad params, network error, malformed upstream response — the element keeps its existing content instead of failing the whole document. Fetches are cached per-URL for five minutes, so a badge rendered repeatedly doesn't re-hit the upstream API on every request.
